# OpenClaw WSL 启动器说明书

## 适用对象

这份说明书用于 Windows 桌面上的 `OpenClaw WSL 启动器`。

它的作用是帮助你在不输入命令的情况下，直接管理 WSL 中的 OpenClaw。

## 交互方式

现在启动器采用“先阅读说明，再确认执行”的方式：

1. 先点击任意功能按钮
2. 右侧会显示该按钮的用途、执行内容和影响范围
3. 阅读完后，再点击：
   - `确认执行 / Confirm`
4. 如果不想继续，点击：
   - `取消待执行 / Cancel`

这样做的目的是避免误操作，尤其是：

- 清理单会话
- 重置默认主会话
- 归档并重置主会话

界面上方按钮区也做了分组：

- `通用运维 / Shared`
- `WebChat 专属 / WebChat`
- `飞书说明 / Feishu`
- `执行控制 / Confirm`
- `帮助说明 / Help`

并新增了模式切换：

- `WebChat / 仪表盘`
- `Feishu / 飞书`

它的意义是：

- 当你主要通过浏览器里的 WebChat / Dashboard 使用 OpenClaw 时，选择 `WebChat`
- 当你主要在飞书里与 OpenClaw 对话时，选择 `Feishu`

这样可以避免把“只作用于 WebChat 本地主会话”的按钮误以为也适用于飞书当前会话。

在 `Feishu / 飞书` 模式下，当前版本新增了一个只读按钮：

- `识别最近飞书会话 / Detect Feishu`

它会扫描最近的会话文件，按飞书相关标记给出候选列表，但仍属于启发式判断，不保证 100% 等于你当前聊天线程。

它的实际用途是：

- 当你感觉飞书里历史太长、越来越卡时，先判断“最可能有问题的是哪条候选会话”
- 它本身不删除任何数据，只负责帮助你判断下一步值不值得归档

候选列表现在会尽量展示更多判断信息，例如：

- 最近活跃时间
- 文件大小
- 命中的飞书标记数量
- 最近用户 / 助手 / 工具活动摘要
- 一小段最近内容预览

这样你能更容易判断哪一条更像你当前正在使用的对话。

当你在候选列表中点选某一项时，右侧说明区会优先显示这条候选的详细摘要，而不是固定模式说明。
你可以直接查看：

- 最近用户说了什么
- 最近助手回复了什么
- 最近工具在执行什么
- 最近几条活动摘要

关闭后的候选默认不会再出现在识别列表里。
如果你想重新查看这些已关闭候选，可以勾选：

- `显示已关闭 / Show Closed`

同时新增了一个高风险按钮：

- `归档最近飞书候选 / Archive Feishu`

它会处理当前评分最高的飞书候选会话，执行前建议先用 `Detect Feishu` 查看候选是否合理。
这个操作会备份索引，并归档候选会话文件，但仍不能保证 100% 就是你当前飞书线程。

在 GUI 中，推荐流程是：

- 先执行 `Detect Feishu`
- 在候选列表中手动点选你怀疑有问题的那一项
- 再执行 `Archive Feishu`

它的实际用途是：

- 当你已经确认“第 1 个飞书候选大概率就是问题来源”时，
- 用它尽量切断这条候选会话继续把旧历史带给大模型的概率

同时新增了一个较温和的按钮：

- `关闭选中会话 / Close Selected`

它会把当前选中的飞书候选从活动索引中移除，但保留原始会话文件。
适合你想先“软关闭”某条候选，观察是否还会继续拖慢大模型，再决定是否做归档。

另外新增了一个通用按钮：

- `安全退出 / Safe Exit`

它会：

- 保存当前会话索引与活动会话文件快照
- 记录退出前状态
- 安全停止 OpenClaw 网关

它不会删除历史，因此下次启动时仍然可以继续当前工作。

如果上一次是通过 `安全退出 / Safe Exit` 结束的，启动器下次打开时会自动显示：

- 上次安全退出时间
- 快照保存位置
- 可恢复提示

## 顶部信息说明

- `WSL: Ubuntu-24.04 | 端口 / Port: 18789`
  - 表示当前启动器操作的 Linux 发行版和网关端口。

- `固定主会话 / Main Session`
  - 当前显示的是本地默认主会话键。
  - 它目前指向 `agent:main:main`。
  - 这不等于“飞书里你当前正在聊天的那个会话”。

- `Session ID`
  - 表示当前主会话的唯一会话编号。
  - 当你要强制续接同一个会话时，可以使用这个 ID。

## 按钮说明

### 1. 启动网关 / Start

**作用**
- 启动 WSL 中的 OpenClaw 网关。

**内部执行**
- 优先使用 systemd 启动：
  - `systemctl --user start openclaw-gateway.service`
- 如果未成功监听 18789 端口，则尝试：
  - `openclaw gateway --force --port 18789`

**影响**
- OpenClaw 网关会在 WSL 中处于运行状态。
- 不会自动打开浏览器。
- 不会自动删除任何历史会话。

### 重启网关 / Restart

**实际使用场景**
- 当网关卡死、模型无响应，或者修改了系统级配置需要生效时。

**作用**
- 相当于强制先停止 (Stop) 再重新启动 (Start)，能解决大部分“僵尸进程”或网络端口占用的问题。

**内部执行**
- 先执行 `openclaw gateway stop`，清理残留进程，然后再执行 `systemctl --user restart openclaw-gateway.service`。

**影响**
- 当前正在进行中的所有对话请求会被打断。
- 已经连接的仪表盘/WebChat 可能会提示断开并自动重连。
- **【不影响】** 之前的聊天历史记录和会话数据均安全保留，不会丢失。

### 2. 打开仪表盘 / Dashboard

**作用**
- 获取当前仪表盘带 token 的地址，并自动在 Windows 浏览器中打开。

**内部执行**
- 运行：
  - `openclaw dashboard`

**影响**
- 会自动打开浏览器页面。
- 不会重启网关。
- 如果网关未启动，可能无法正常打开控制页面。

### 3. 查看状态 / Status

**作用**
- 查看 OpenClaw 当前状态摘要。

**内部执行**
- 优先执行：
  - `timeout 12 openclaw status`
- 若超时则降级执行：
  - `openclaw health`

**影响**
- 只读操作，不改配置。
- 不会删除会话。
- 不会打开浏览器。

### 4. 健康检查 / Health

**作用**
- 检查网关与运行环境是否健康。

**内部执行**
- 运行：
  - `openclaw health`

**影响**
- 只读操作。
- 不会修改会话，不会启动浏览器。

### 5. 停止网关 / Stop

**作用**
- 停止 WSL 中运行的 OpenClaw 网关。

**内部执行**
- 依次尝试：
  - `openclaw gateway stop`
  - `pkill -f clawdbot`
  - `pkill -f openclaw`

**影响**
- 仪表盘会失去连接。
- 不删除历史数据。
- 不影响配置文件。

### 6. 清空日志 / Clear

**作用**
- 清空当前启动器窗口中的日志显示。

**内部执行**
- 只清空界面文本框。

**影响**
- 仅影响当前窗口显示。
- 不删除 WSL 里的任何日志文件。

### 7. 清理单会话 / Prune

**作用**
- 清理多余会话，只保留主会话。

**内部执行**
- 调用：
  - `clean_openclaw_sessions.py`

**影响**
- 会备份原 `sessions.json`
- 会把会话索引缩减为：
  - `agent:main:main`
  - `sessions`
- 有助于减少多会话干扰
- 不会删除主会话本身

**注意**
- 这个操作主要是清理会话索引，不是删除全部聊天文件。
- 某些运行过程仍可能以后再次生成新的 subagent / cron 会话。

### 8. 一键启动并打开 / Launch

**作用**
- 一次性完成“启动网关 + 打开仪表盘”。

**内部执行**
- 先执行 `Start`
- 再执行 `Dashboard`

**影响**
- 会自动拉起网关
- 会自动打开浏览器
- 适合日常直接使用

### 9. 重置默认主会话 / Reset Main

**作用**
- 只重置本地默认主会话 `agent:main:main`。
- 适合怀疑默认主会话上下文太长，但还不想移动旧会话文件时使用。

**内部执行**
- 调用：
  - `manage_openclaw_main_session.py --mode reset`

**影响**
- 会备份 `sessions.json`
- 会从索引中移除 `agent:main:main`
- 不会移动旧的主会话 jsonl 文件
- 下次本地主会话会更像重新开始

**注意**
- 这是“本地默认主会话”的重置，不保证直接影响飞书当前线程会话。

### 10. 归档并重置主会话 / Archive Main

**作用**
- 归档本地默认主会话，并让下一次本地主会话从干净状态开始。
- 适合怀疑主会话本身已经很长、很重、容易卡住时使用。

**内部执行**
- 调用：
  - `manage_openclaw_main_session.py --mode archive`

**影响**
- 会备份 `sessions.json`
- 会把当前主会话的 jsonl / lock 文件移动到备份目录
- 会从索引中移除 `agent:main:main`
- 对减少“本地默认主会话上下文负担”更有效

**注意**
- 这仍然不是对飞书当前会话的精确重置，而是对本地默认主会话的重置。

### 11. 按钮说明 / Help

**作用**
- 在应用右侧展示这份说明书。

**内部执行**
- 读取：
  - `OpenClaw-WSL-Launcher-Guide.md`
- 并显示在应用右侧说明面板

**影响**
- 不改任何配置
- 不影响网关状态

### 12. 确认执行 / Confirm

**作用**
- 执行当前“待执行动作”。

**内部逻辑**
- 只有在你先点击某个功能按钮之后，这个按钮才有意义。
- 它会执行当前右侧说明对应的那个动作。

**影响**
- 会真正执行按钮命令。
- 如果没有待执行动作，则不会做任何实际操作。

### 13. 取消待执行 / Cancel

**作用**
- 取消当前待执行动作。

**内部逻辑**
- 清空当前待执行状态
- 右侧说明恢复为总说明

**影响**
- 不会执行任何命令
- 不会修改网关、会话或配置

## 推荐使用顺序

### 日常启动

直接点击：

- `一键启动并打开 / Launch`

### 网页打不开时

依次点击：

- `启动网关 / Start`
- `打开仪表盘 / Dashboard`

### 会话变多时

点击：

- `清理单会话 / Prune`

### 主会话太长、模型容易卡住时

优先尝试：

- `重置默认主会话 / Reset Main`

如果仍然太重，或你明确想把旧主会话归档后再从新状态开始：

- `归档并重置主会话 / Archive Main`

### 想确认是否正常运行

点击：

- `查看状态 / Status`
- `健康检查 / Health`

## 会发生什么，不会发生什么

### 会发生的事

- 会调用 WSL 中的 OpenClaw 命令
- 会在需要时自动打开浏览器
- 会在点击 `Prune` 时清理会话索引
- 会在点击 `Reset Main / Archive Main` 时重置本地默认主会话索引

### 不会发生的事

- 不会自动删除你的主会话
- 不会自动删除配置文件
- 不会自动卸载 OpenClaw
- 不会自动清空真正的历史日志文件
- 不保证精准重置你飞书中当前正在聊天的那个线程会话

## 项目结构
- `OpenClaw-WSL-Launcher.ps1`
  - 核心 PowerShell 脚本，包含所有 GUI 逻辑和向 WSL 发送命令的实现。
- `OpenClaw-WSL-Launcher.bat`
  - Windows 下的双击启动入口，自动调用 `.ps1`。
- `clean_openclaw_sessions.py`
  - 用于清理 WebChat 多余会话。
- `manage_openclaw_main_session.py`
  - 用于重置/归档 WebChat 主会话。
- `detect_feishu_sessions.py`
  - 飞书模式下的启发式会话识别脚本。
- `manage_feishu_candidate.py`
  - 飞书模式下关闭或归档候选会话的脚本。
- `safe_exit_openclaw.py`
  - 安全退出时生成状态快照的脚本。
- `OPENCLAW_WSL_RUNBOOK.md`
  - 运维与排错指南。
